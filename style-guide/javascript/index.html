JavaScript

Linting

  • We follow Airbnb’s JavaScript style guide with ESlint integration.
  • In Vue we use @vue/cli-plugin-eslint and @vue/eslint-config-prettier.

Formatting

  • We use Prettier and we configure it for works well with ESlint.

Installing dependencies

First we install ESlint + Prettier.

npm install -D eslint prettier

Then we install Airbnb config. Airbnb config has own dependencies like:

eslint-config-airbnb eslint eslint-plugin-jsx-a11y
eslint-plugin-import eslint-plugin-react

instead of installing all these manually. We use:

npx install-peerdeps --dev eslint-config-airbnb

(For using npx you need to use npm 5+ )

After we install: eslint-config-prettier for disabling formatting for ESLint and eslint-plugin-prettier for allowing ESLint to show formatting errors.

npm install -D eslint-config-prettier eslint-plugin-prettier

Configuration

Our default .eslintrc file:

{
  "extends": ["airbnb", "prettier"],
  "plugins": ["prettier"],
  "parserOptions": {
    "sourceType": "module",
    "ecmaFeatures": {
      "jsx": true
    }
  },
  "env": {
    "browser": true,
    "es6": true,
    "node": true
  },
  "rules": {
    "prettier/prettier": "error"
  }
}

Our default .prettierrc file:

{
  "printWidth": 100,
  "singleQuote": true,
  "trailingComma": "all"
}

In package.json file:

{
  ...
  "scripts": {
      "lint": "eslint ."
  },
  ...
}

VS Code

We use Prettier and ESLint extensions.

And make sure to Prettier runs when on save. In user settings we set:

"[javascript]": {
    "editor.formatOnSave": true
}
